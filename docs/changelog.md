# 📝 版本更新记录

## v1.1.8

- **🏗️ 架构优化**
  - 重构本地进度数据库的连接生命周期，采用「即用即开、即用即关」的细粒度管理模式。仅在数据读写的极短时间内持有数据库锁，歌曲播放、下载及待机等待等绝大多数场景均释放文件锁，从根源上降低多账号并发、多实例运行时的锁冲突概率。

- **🐞 Bug 修复**
  - 修复 `playids` 播放任务因数据库锁异常中断的问题，新增锁冲突降级机制，保障任务稳定运行。

## v1.1.4

- **✨ 新增功能**
  - 新增乐迷团自动化任务 `ncmm fansgroup`，支持独立命令执行，也可通过批量任务入口 `ncmm task --fansgroup` 或配置项自动触发。
  - 任务会根据配置筛选主账号/辅助账号，自动查询乐迷团任务完成进度，并按剩余次数补齐任务。
  - 实现播放、图文发布、分享、点赞全类型乐迷团任务自动化，图文内容可依据配置规则自动删除。

- **🏗️ 架构优化**
  - 日常签到模块瘦身解耦与服务化重构
    - 将原本臃肿庞大的 `sign` 核心业务进行模块化拆分，解耦为职责单一的细分子服务（如黑胶 VIP 任务、云贝中心日常任务），大幅提升代码可维护性与扩展性。
    - 重新定位 `sign` 为普通账号日常签到，彻底移除其内所有跟音乐人相关的任务逻辑与账号身份校验，使业务边界更加清晰。
  - 音乐人模块全局重命名与重构
    - 将原 `musician-vip` 模块与命令全局更名为 `musician`。
    - 将日常音乐人任务（日常签到与云豆领奖，从原 `sign` 中拆出）与 VIP 进阶任务统一规范为两阶段顺序执行。
    - 将原本位于 `sign` 节点下的音乐人身份缓存参数 `identityCacheDays` 迁移并移至 `musician` 控制参数下，实现配置概念的深度一致。
  - 配置文件自动向下兼容迁移
    - 启动时若读取到旧版配置文件，程序会自动迁移重构，提供零感平滑升级体验。

- **💫 体验优化**
  - 任务开关与跳过状态日志精细化
    - 确保即便所有任务开关全部为 `false`，也能正常打印并告知每个任务的跳过状态。
    - 差异化提示跳过来源：在批量执行（`task`）时，精确区分任务是因为“命令行未指定”还是“配置文件已关闭 (task.xxx = false)”。
    - 增加账号级与配置级细分跳过提示：当单独执行子任务（`musician`/`sign`/`playids`）因 `enableMain`、`enableSecondaries` 关闭或对应账号未配置时，在控制台打印详实的提示日志，方便快速定位未执行缘由。

- **🐞 Bug 修复**
  - 移除 note.go 中冗余的自定义解析逻辑，统一依托底层 Cookie 加载机制，修复编译异常报错。

## v1.1.3
- **✨ 新增功能**：
  - 新增黑胶 VIP 任务打卡总开关 `enableVipTask`，配置文件中默认开启，支持多账号轮询。
  - 新增一整套黑胶 VIP 会员日常任务自动化链路，可一键打卡并自动申领成长值：
    - 黑胶乐签打卡：对接底层 EAPI 打卡接口，并重构任务状态判定逻辑，前置拦截签到流程，跳过列表已完成过滤，解决因服务端服务端缓存误判导致的漏签、签到失效问题。
    - 红心 3 首 VIP 歌曲：随机拉取热门 VIP 歌单执行红心操作，经过随机时延后自动取消红心，在满足打卡要求的同时，避免改动用户原有收藏歌单。
    - 浏览停留模拟：支持查看AI调音大师、浏览云贝中心、分享单曲到站外等跳转行为，模拟客户端真实停留时长。
    - 免费领福利打卡：对接新版权益中心 EAPI，自动拉取常驻免费商家券池并领取首张优惠券，完成福利打卡，适配当前版本规则。
  - 任务执行完毕后，自动调用 EAPI 一键申领接口，统一领取黑胶 VIP 任务对应的全部成长值奖励。

- **💫 体验优化**：
  - 请求参数脱敏：移除接口中硬编码的设备和隐私字段。
  - 日志友好回显：移除模拟页面加载日志中冗长的跳转链接，精简控制台输出内容。

## v1.1.2

  - 播放任务支持高容错运行，单源加载失败不影响整体任务执行。
  - 新增网络超时限制，防止无效链接导致程序卡死。
  - 完善笔记发布图片多源与智能探测机制，支持本地图片、直链及包含链接的文本列表。

  - 配置文件无感自动升级：启动时自动兼容旧版配置字段，完整保留文件格式与用户自定义注释。
  - 云贝预约领奖流程优化：支持在奖励领取成功后自动检查并进行新活动预约。
  - 优化日推模拟听歌任务：将后台并发播放改为前台串行执行，放在账号任务的最后执行，避免多账号间日志交错混乱。
  - 优化云贝领奖流程：在领取完云贝任务奖励后，再次自动获取并展示该账号下各任务的最终完成状态。

## v1.1.0
- **✨ 新增功能**：
  - 上线音乐人多账号任务套件：新增 `ncmm sign` 命令，支持主 / 辅助账号自动轮询，批量完成多端签到与任务领取，涵盖音乐人签到、云豆 / 云贝签到、黑胶 VIP 乐签等场景。

  - 日志体系优化：清理冗余底层日志，重新划分日志分类，提升内容可读性。

## v1.0.0
- **✨ 新增功能**：
  - 完成项目基础架构搭建：基于 Go 语言落地核心框架，搭建模块化目录结构。
  - 支持多方式账号登录：包含扫码登录、手机号密码 / 短信验证码登录、批量导入 Cookie、CookieCloud 自动同步。
  - 实现歌曲模拟播放能力：通过 `playids` 命令解析流媒体地址、模拟播放并上报听歌数据，支持自定义随机等待间隔。
  - 播放限额与进度存储：集成本地嵌入式数据库，可设置每日播放下限，运行达标后自动停止，规避平台风控。
  - 全局自定义 UA 配置：支持在配置文件统一设置请求标识，提升请求隐蔽性。
- **💫 体验优化**：
  - 新增 CDN 音频缓存：首次播放从 CDN 加载音频流，同次运行内重复播放读取跳过下载，减少带宽消耗。
  - 优化播放休眠逻辑：细化进度展示与静默等待的计算规则，运行表现更自然。
